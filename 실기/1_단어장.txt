 # 데이터베이스.

DBA : 데이터베이스 관리자.
DBMS : 데이터베이스 관리 시스템.

데이터베이스 : 자료( Data )의 모임.
데이터베이스 설계 : 요구조건 분석 -> 개념적 설계 -> 논리적 설계 -> 물리적 설계 -> 구현.

스키마 : 데이터베이스 전반적인 명세.
외부스키마 : 개인적인 관점 /  서브 스키마.
개념스키마 : 조지적 관점 / 논리 스키마.
내부스키마 : 시스템 프로그래머의 관점.

Degree( 차수 ) : 속성의 개수. 
Cardinality( 기수 ) : 튜플의 개수.

트랜잭션 : 하나의 작업 수행을 위한 연산들의 집합.
SQL : 데이터베이스 질의어.

DDL : 데이터베이스 정의( Definition ) 언어. ( Create, Alter, Drop, Tuncate )
DML : 데이터베이스 조작( Manipulation ) 언어. ( Select, Insert, Update, Delete ) 
DCL : 데이터베이서 제어( Control ) 언어. ( Grant, Revoke, commit, rollback ) 

alter table 테이블명 ADD 컬럼명 데이터타입 ; 
alter table 테이블명 modify 컬럼명 데이터타입 ; 
alter table 테이블명 drop 컬럼명 ;

select 컬럼명 from 테이블명 where 조건 group by 조건 having 조건 order by 정렬방식 ; 
select distinct 컬럼명 from 테이블명 ;

drop table 테이블명 where 조건.
drop table 테이블명 cascade - 참조시 같이 삭제.
drop table 테이블명 restrict - 참조시 취소.

create view 뷰이름 as select 컬럼 from 테이블명 where 조건 ;
select 컬럼 from 테이블명1 join 테이블명2 on 테이블명1.컬럼 = 테이블명2.컬럼 ;

union : 합집합( 중복 제거 )
union all : 합집합( 중복 표시 )
intersect : 교집합.
minus : 차집합.

기본키 : 후보키 중 선정된 키.
후보키 : 유일성 만족, 최소성 만족.
슈퍼키 : 유일성 만족, 최소성 불만족.
대체키 : 기본키로 선택되지 못한 후보키.
외래키 : 다른 테이블의 행 식별 키.

Anomaly : 데이터베이스 이상현상

관계 대수 : 정보 유도 절차적 언어.
관계 해석 : 정보 명시 비절차적 언어.



 # 운영 체제.

운영체제 Operating System : 컴퓨터 하드웨어와 사용자 간에 위치, 하드웨어 소프트웨어 자원 등을 관리.

윈도우( Windows ) : 선점형멀티테스킹, GUI, PNP, NTFS
도스( DOS ) : CUI, CLI
유닉스( Unix ) : 다중 사용자, 멀티 태스킹, CUI.
리눅스 : 오픈 소스 운영체제.

PNP : Plug & PLay.
핫 스왑 : 시스템 전원이 켜진 상태에서, 외부 장치를 안전하게 연결 및 제거하는 기능.
핫 플러그 : 새로운 장치를 연결할 시 시스템 전원이 꺼진 상태에서 사용하는 기능.

Unix.
아이노드( i-node ) : Unix의 정규파일, 디렉토리 등 파일 시스템에 관한 정보 보유.
심볼릭 링크, 소프트 링크 : Unix의 바로가기.
하드 링크 : Unix의 원본과 동기화된 바로가기. ( 링크파일 삭제 시 원본 삭제 )
UFS 파일 시스템 : EXT2 - UFS에서 불필요한 구조들을 제거한 유닉스 파일 시스템.
	           EXT3 - EXT2에서 저널링 기능을 추가한 유닉스 파일 시스템.

Windows.
FAT12, FAT16, FAT32 파일 시스템 : 연결리스트 형태의 자료 구조 
NTFS 파일 시스템 : FAT과 HPFS의 단점을 개선.

가상화 : 단일 호스트에서 다수의 서로 다른 운영체제를 구동 지원하는 기능.
하이퍼바이저 : 단일 호스트에서 다수의 운영체제를 가상으로 구동 지원하는 플랫폼.

윈도우 대표 단축키.
Win + E : 윈도우 탐색기 실행.
Win + R : 윈도우 실행창 실행.
Win + F : 찾기.
Win + D : 바탕화면 표시.
Win + L : 사용자 전환, 윈도우 잠금.
Win + Pause : 시스템 구성 요소 확인.

윈도우 도스 명령어.
경로 변경 : cd
목록 출력 : dir.
구조 복사 : xcopy.
디렉토리 생성 : mkdir.
하위파일 삭제 : del.
속성 설정 : attrib.

리눅스 명령어.
경로 변경 : cd
목록 출력 : ls.
구조 복사 : cp
디렉토리 생성 : mkdir.
하위파일 삭제 : rm
속성 설정 : chmod.



 # 애플리케이션 테스트.

결함 : 프로그램과 명세서 간의 차이, 업무 내용 불일치.
결함 심각도 : High - 프로세스 진행 불가능. ( 시스템 다운 )
                 Medium - 시스템 흐름에 영향을 미침 ( 보안 관련 오류 )
                 Low - 상황에 맞지 않는 결과 및 화면구성 결함. ( 에러 메세지 미출력 )

결함 검사.
Fixed : 결함 수정.
Assigned : 결함 할당.
Open : 결함 보고( 분석 전 )
Closed : 수정 후 결함 미발견.
Deferred : 수정 연기.
Clarified : 비결함.

결함 관리 프로세스 7과정.
관리계획 -> 기록 -> 검토 -> 수정 -> 재확인 -> 상태추적 -> 최종분석.

에러 : 개발 중 발생한 부정확한 결과.
오류 : 프로그램 버전간의 차이로 발생.
실패 : 프로그램 버전간의 실행 결과의 차이.
휴먼에러 : 원인이 인간의 실수인 에러.

코드 인스펙션 : 자동화 도구 사용, 결함 발견, 수정.
워크 스루 : 코드 품질 평가, 개선 목적 검토.
살충제 패러독스 : 동일한 테스트의 비정상적인 결함 검수.
테스트 케이스 : 요구사항 준수 여부 확인용 입력값.

스텁 : 하위 모듈 테스트 진행.
드라이버 : 상위 모듈 테스트 진행.

V-모델 : 소프트웨어 개발 프로세스인 폭포수 모델의 확장된 형태.

유스케이스 : 사용자 시스템 동작 시나리오.
유스케이스 다이어그램 : 시스템 간 상호작용 표현.

블랙박스 테스트 : 사용자 관점, 명세 기반.
화이트박스 테스트 : 개발자 관점, 내부구조 & 동작 검사.

테스트 오라클 : 사전 정의된 참 값을 대입 비교.
참 오라클 : 기대 결과 생성 후 오류 검출.
샘플링 오라클 : 특정 입력 값의 기대 결과 제공.
휴리스틱 오라클 : 확률-직관에 의한 예상 결과.
일관성 검사 오라클 : App 변경 전후의 값 동일 여부 검증.

소프트웨어 아키텍처 : 소프트웨어의 골격이 되는 기본 구조.

Java : 썬 마이크로시스템즈 개발 객체 지향 프로그래밍 언어.
C언어 : B언어에서 파생된 프로그래밍 언어.
xUnit : 테스트 프레임 워크.
JSON : XML의 단점을 보완한 JSP 기반 독립형 언어.



 # 네트워크.

IPv4 : 32비트, 유니/멀티/브로드캐스트, ' . '로 구분.
IPv6 : 128비트, 멀티/애니/유니캐스트, ' : '로 구분.

OSI 7계층 : 네트워크 통신을 계층으로 나눈 참조모델.
물리 계층, 데이터 링크 계층, 네트워크 계층, 
전송 계층, 세션 계층, 표현 계층, 응용 계층.

TCP/IP 4계층 : 인터넷 정보교환 프로토콜 계층.
네트워크 액세스계층( 1~2 ), 인터넷 계층( 3 ), 전송계층( 4 ), 응용 계층( 5~7 )

허브 : 하나의 대역폭 분배 장치 - 물리계층.
리피터 : 네트워크 전송 신호 증폭 - 물리계층.
라우터 : 네트워크 상 최적의 경로 제공 - 네트워크 계층.
브리지 : 같은 구조 네트워크 연결 - 데이터링크 계층.
게이트웨이 : 다른 구조 네트워크 연결 - 세션계층 ( 전 계층 사용 )

프로토콜 기본요소.
1. 구문 : 데이터 형식 규정.
2. 의미 : 오류 제어 정보 규정.
3. 시간 : 속도 조정 규정.

TELNET : 원격 통신 프로토콜.
SSH : 원격 통신 프로토콜
ARP : 논리주소로 물리주소 검색.
RARP : 물리주소로 논리주소 검색.

TCP : 데이터 (검수)전용 프로토콜.
UDP : 데이터 (비검수)전용 프로토콜.
FTP : 파일 전송 프로토콜.
HTTP : 웹 데이터 전송 프로토콜.

SOAP : 서로 다른 기기 간 정보통신.
VLAN : 브리지를 이용한 복수 논리적 네트워크.
디스패치 : 준비에서 실행으로 프로세스 상태 전이.
IOT : 인터넷을 포함한 사물 인터넷.
XML : 높은 호환성의 웹 프로그래밍 언어.

노드 : 단말 접합점.
애드훅 네트워크 : 노드에 의해 자율적으로 구성된 기반 구조 없는 네트워크.
