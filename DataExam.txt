# 스프레드시트의 개념.


컴퓨터를 이용하여 각종 계싼 관련 업무를 처리하는 전자 계산장을 의미.

수학적 함수나 통계 처리와 같은 수치 자료 계산을 셀에 입력하여 계산,검색 등을 빠르게 처리하는 소프트웨어.

ex ) MS-EXCEL, 훈민시트, 로터스, 쿼트로프로, 멀티플랜 등....



# 스프레드시트의 기능.


데이터의 입력과 수치 데이터의 계싼 기능 및 데이터가 변경되면 자동으로 재계산하는 기능.

차트 작성 기능과 문서 작성 기능. ( 워드프로세서 기능 )

입력 데이터를 이용한 데이터 활용 기능. ( 데이터베이스 관리 기능 )

그림, 클립아트, 지도와 같은 다양한 개체 삽입 기능.

반복적인 작업을 처리하는 매크로 기능.

슬라이드 쇼 기능은 없다!!

# 운영체제( Operating System)의 정의.


컴퓨터의 성능을 효율적으로 운영, 관리, 감독하기 위한 시스템 프로그램.

컴퓨터 하드웨어와 사용자 간의 인터페이스를 담당하는 시스템 프로그램.

ex ) MS-DOS, ZENIX, UNIX, LINUX, Windoes 등.



# 운영체제의 기능.


초기작업 환결 설정 기능.

프로세스 관리.

주기억 장치, 처리기, 주변 장치 등의 자원 관리 기능.

입출력 관리 및 파일 관리 기능.

사용자에게 편의성 제공.

CPU 스케줄링 ***

기억 장치의 할당 및 회수.

하드웨어 장치와 프로그램수행 제어.



# 운영체제의 성능 평가 요소 ( 운영 체제의 목적 )


처리능력 ( Throughput ) 향상 : 시스템의 생산성을 나타내는 단위로, 일정 시간 동안 처리하는 일의 양.

응답시간 (Turnaroung Time ) 단축 : 작업 의뢰 후 시스템에서 결과가 얻어지는 시간 단축.

사용가능도 ( Availability ) 증대 : 시스템을 얼마나 빠르게 사용할 수 있는가의 정도.

신뢰도 ( Reliability ) 향상 : 주어진 문제를 얼마나 정확하게 처리하는가의 정도.



# 운영체제의 구성.


1. 제어 프로그램 ( Crontrol Program )

- 감시 프로그램 ( Supervisor Program ) : 시스템 전체의 동작 상태를 감독하고 지원.

- 작업관리 프로그램 ( Job Management Program ) : 작업을 처리하고 다른 작업으로 이행을 위한 처리.

- 데이터 관리 프로그램 ( Data Management Program ) : 데이터 전송, 파일 조작 및 처리 등.


2. 처리 프로그램 ( Process Program )

- 언어 번역 프로그램 ( Language Translator Program ) : 원시 프로그램을 기계어로 변환 시키는 프로그램.

- 서비스 프로그램 ( Service Program ) : 시스템에서 사용 빈도가 높은 프로그램을 미리 개발하여 놓은 프로그램.

- 문체 처리 프로그램 ( Problem Processing Program ) : 컴퓨터 사용자가 필요한 업무에 맞게 개발한 프로그램.



# 언어 번역 과정.


원시프로그램 -> 언어번역기 -> 목적프로그램 -> 링커 -> 로드모듈 -> 로더 -> 실행 -> 결과.


로드모듈 : 실행 가능한 상태의 프로그램.

로더 : 로드 모듈 프로그램을 주기억 장치 내로 옮겨서 실행해 주는 소프트웨어.



# 운영체제의 운영 방식.


1. 일괄 처리 시스템 ( Batch Processing System )

- 처리할 데이터를 한꺼번에 모아 일정한 시간이 경과되거나 일정한 양이 되었을 때 처리하는 방식.

ex ) 월 급여 처리, 연말 정산 처리.


2. 실시간 처리 시스템 ( Real Time Processing System )

- 데이터가 발생되는 즉시 처리하는 방식으로, 바로 응답을 받아볼 수 있는 시스템.

ex ) 승차권 예약, 좌석 예약, 온라인 업무, 로봇 제어 등.


3. 다중 프로그래밍 시스템 ( Multi - Programming System )

- 한 대의 컴퓨터 메모리에 여러 개의 프로그램을 동시에 적재 한 후 실행하는 방식.

- Idle Time ( 유휴 시간 ) 을 줄여 시스템의 사용 효율을 높임.


4. 시분할 처리 시스템 ( Time Sharing Processing System )

- 한 대의 컴퓨터를 동시에 여러 명의 사용자가 대화식의 방식으로 사용하는 시스템.

- 모든 사용자들에게 CPU가 일정한 시간대로 나누어서 하나의 작업에게만 자신의 권한을 할당해 주는 방식.

ex ) 라운드 로빈 스케줄링.


5. 다중 처리 시스템 ( Multi - Processing System )

- 한 대의 컴퓨터에 CPU가 두 개 이상 설치되어 여러 작업을 병행 처리하는 방식.

- 여러 작업을 동시에 처리하게 함으로써 시스템 처리 능력을 향상시킴.


6. 분산 처리 시스템 ( Distributed Processing System )

- 분산된 여러 컴퓨터를 네트워크로 연결하여 처리하는 시스템.

# 교착 상태 ( Deadlock )


자원은 한정되어 있으나 각 프로세스들이 서로 자원을 차지하려고 무한정 대기하는 상태.



# 교착상태가 일어나기 위한 4가지 조건.


상호배제 Mutual Exclusion : 각각 프로세스가 배타적 통제권을 요구.

점유와 대기 Hold and Wait : 점유하면서 다른 자원을 요구.

비선점 NoN-Preemption : 프로세스에 할당된 자원을 사용이 끝날 때까지 강제로 빼앗을 수 없을 때.

환형 대기 Circular Wait : 각각 프로세스 간 자원의 요규가 순환되는 원형과 같은 형태.



# 교착 상태 해결 방안.


예방 Prevention , 회피 Avoidance , 탐지 Detection , 복구 Recovery .



# 스케줄링 Scheduling


컴퓨터의 자원을 보다 효율적으로 이용하기 위해 작업 순서와 시간을 할당하는 것.


1. 선점형 ( Preemptive ) 기법.

- 하나의 프로세스가 CPU를 점유하고 있을 때 다른 프로세스가 CPU를 빼앗아 차지할 수 있는 방법.

- RR 스케줄링 ( Round Robin ) : 차례대로 일정한 시간 할당량 동안 처리기를 차지하는 기법.

- SRT 스케줄링 ( Shortest Remaining Time ) : 남은 처리 시간이 가장 짧은 프로세스를 우선 처리.

- MFQ 스케줄링 ( Multilevel Feedback Queue ) : 작업을 여러단계로 나누어 처리하는 방식.


2. 비선점형 ( Non - Preemptive ) 기법.

- 특정한 프로세스가 작업이 끝날때까지 CPU를 독점하는 방법.

- FIFO 스케줄링 : 먼저 들어온 것을 우선 처리.

- Priority 스케줄링 : 각각의 작업에 우선순위를 정하여 처리.

- SJF 스케줄링 ( Shortest Job First ) : 작업 시간이 짧은 것부터 먼저 처리.

- HRN 스케줄링 ( Highest Response ratio Next ) : SJF에서 + 대기시간이 오래 된 것.
