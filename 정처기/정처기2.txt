# DOS Disk Operating System의 개요.


개인용 컴퓨터( PC )에서 디스크 관리 및 파일 관리 등을 다루기 윟나 운영체제.

CUI 방식이다.




# DOS의 기능.


컴퓨터와 사용자 사이에서 인터페이스 역할.

사용자가 입력한 명령어를 처리하며 디스크 및 파일 관리를 담당.

입출력 관리, 메모리 할당, 주변 장치 관리, 인터럽트 처리 등을 제어.




# DOS 부팅


1. 부팅 순서.

- 본체 모니터 등 주변 기기의 전원을 킨다.

- ROM-BIOS에 의해 기기의 이상 유무를 점검.

- IO.SYS 파일을 읽어 주기억 장치에 적재.

- MSDOS.SYS 파일을 주기억 장치로 읽어 들임.

- CONFIG.SYS 파일을 주기억 장치로 읽어 들임.

- COMMAND.COM 파일을 주기억 장치로 읽어 들임.

- AUTOEXEC.BAT 파일을 실행.


2. 부팅 종류.

- 콜드부팅 : 하드웨어적인 부팅으로, 리셋버튼을 눌러 재부팅하는 방법.

- 웜 부팅 : 소프트웨어적인 부팅으로, CTRL + ALT + DELETE를 동시에 눌러 재부팅.




# 시스템 파일 System File.


IO.SYS : 실제적인 입출력을 수행. ( 숨김 속성 )

MSDOS.SYS : 시스템 호출, 파일관리, 메모리관리, 프로세서관리, 하드웨어 관리를 담당. ( 숨김 속성 )

COMMAND.COM : 명령어 해석 및 명령어 처리기 기능. 내부 명령어 포함.



# 파일 File.


프로그램이나 자료를 저장할 때 사용되는 저장 단위. 그 종류나 기능에 따라 구분하고 파일명과 확장자로 구성됨.


DOS 파일명의 특징.

1. 파일명은 영문 8자( 한글 4자 ), 확장자는 3자 ( 한글 1자 )로 구성.

2. 공백 포함 X.

3. 특수문자 사용 불가.


만능문자 2가지.

1. ?

2. *




# DOS 명령어 구분.


1. 내부 명령어.

- DOS 부팅시 COMMAN.COM이 실행될 때 주기억 장치에 상주하여 키보드를 통해 명령이 입력되면 바로 실행

되는 명령어.


2. 외부 명령어.

- 내부 명령어보다 사용 횟수가 비교적 적고 독립적인 파일이 존재함.

- 보조 기억 장치에 명령어를 저장시켜 두었다가 명령 실행시 주기억 장치로 가져와 실행.


3. 디스크 관련 명령.

- FORMAT : 디스크에 저장이 가능하도록 트랙과 섹터를 형성하여 초기화 작업을 수행.

- FDISK : 하드 디스크 파티션의 논리적 분할과 삭제 작업을 수행.

- CHKDSK : 디스크의 상태를 점검하고 손상된 부분을 복구.


4. 디렉토리 관련 명령.

- MD : 새로운 디렉터리를 생성하는 명령.

- RD : 기존에 생성되어 있는 디렉터리를 삭제하는 명령.

- XCOPY : 디렉토리내에 있는 모든 것을 복사. ( 숨겨진 파일과 시스템 파일은 복사 불가 )


5. 파일 관련 명령.

- DIR : 디스크 내의 파일 목록, 파일 정보, 파일 수, 파일 크기, 생성날짜와 시간 등을 표시하는 내부 명령어.

- TYPE : 모든 텍스트 파일의 내용을 표시하는 내부 명령어.

- ATTRIB : 파일의 속성을 지정 및 해제하는 명령어.


6. 데이터 가공 명령.

- FIND : 여러 개의 파일에서 특정한 문자열을 검색하는 기능.




# 환경 설정 파일 ( CONFIG.SYS )


시스템 환경을 설정해 주는 파일, 표준 장치 외 주변 장치에 대한 기본 환경 설정 가능.

디스크의 동작 속도를 향상시켜 주는 버퍼 및 캐시의 설정이 가능.

루트 디렉터리에 존재해야 함.

# Windows의 특징.


1. 선점형 멀티태스킹 지원 : 여러 프로그램을 동시에 실행할 수 있으며, 문제가 있는 프로그램을 강제 종료 가능.

2. Plug & Play 지원 : 새로운 하드웨어 설치 및 설정을 자동으로 처리.

3. GUI : 사용자와 컴퓨터 간의 인터페이스를 GUI로 함.

4. 긴 파일명 지원. ( VFAT - DOS는 FAT 방식 )

5. 강화된 멀티미디어 지원.



# Windows의 종료


1. 시작 - 시스템 종료.

2. ALT + F4



# 바로 가기 키.


F1 : 도움말 실행.

F2 : 이름 바꾸기.

F3 : 찾기 ( 파일 , 폴더 )

F4 : 주소 표시줄 열기.

F5 : 새로움 정보로 고침.

F6 : 탐색기의 왼쪽과 오른쪽 구역 사이의 이동.

F8 : WINDOWS 부팅시 부팅 메뉴가 나타남.


CTRL + ESC : 시작 메뉴 호출.

SHIFT + F10 : 프로그램이나 선택한 항목의 바로 가기 메뉴 표시.




# 작업 표시줄 ( TASK BAR )


현재 실행중인 프로그램들이 단추 형식으로 표시되어, '시작' 단추와 시스템 트레이 등이 표시됨.

바탕 화면에서 마우스 조작만으로 자유롭게 이동이 가능, 화면의 최대 50%까지 조절가능.




# 바로 가기 아이콘 ( 단축 아이콘 )


자주 사용하는 프로그램을 보다 빠르고 편리하게 실행시킬 수 있음.

모든 개체에 대해 바로 가기 아이콘을 만들 수 있으며, 여러 개 작성이 가능함.

바로가기 아이콘을 삭제해도 원본에 영향 없음.

확장자 : .LNK




# 보조프로그램.


1. 메모장.

- WINDOWS에서 제공하는 가장 기본적인 문서 편집기.

- OLE 기능을 지원하지 않아 그림판, 워드패드 등에서 작업한 것을 연결하지 못한다.

- 확장자 : .TXT




# 파일 FILE.


하나의 폴더 내에 동일한 이름의 파일이 존재할 수 없음.

컴퓨터에서 사용되는 자료 저장의 기본 단위.




# 폴더 FOLDER.


DOS의 디렉토리와 같은 개념으로, 서로 관련있는 파일들을 저장하는 장소로 파일들을 효율적으로 관리.


1. 찾기

- 파일의 이름, 종료, 크기 등을 지정하여 찾을 수 있음.

- 단축키 : F3을 눌러서 실행.

- '찾기' 대화 상자의 '찾기' 탭 : 이름 및 위치, 날짜, 고급 등을 이용해서 찾을 수 있다.


2. 복사와 이동.

- 복사, 이동, 붙여넣기를 할 때에는 클립보드를 사용.




# WINDOWS 탐색기.


'내 컴퓨터' 의 모든 기능을 수행함.

최상위 위치는 '내컴퓨터' 보다 한 단계 위인 '바탕 화면' 임.


1. 파일 및 폴더 선택.

- 한 개의 데이터만 선택 : 클릭.

- 불연속적인 데이터를 선택 : CTRL + 클릭.

- 연속적인 데이털르 선택 : SHIFT + 클릭.

- 전체 데이터를 선택 : CTRL + A.




# 휴지통 RECYCLED BIN


삭제된 항목을 임시로 보관함으로써 실수에 대비하도록 하는 WINDOWS 내의 임시보관장소.

'휴지통비우기'를 하면 완전히 삭제됨.


1. 휴지통으로 복구할 수 없는 경우.

- SHIFT + DELETE로 삭제한 파일.

- DOS 모드에서 삭제한 파일.

- 플로피 디스크나 네트워크 드라이브 상에서 삭제한 파일.

- 휴지통 비우기를 한 파일 및 폴더.

- 휴지통 설정을 ' 즉시 제거 ' 로 바꾼 경우.




# 시스템 최적화 관리.


1. 디스크 검사.

- 잃어버린 클러스터, FAT, 오류 등 디스크의 논리적인 오류 및 디스크 표면의 오류나 불량 섹터를 검사.

- 정기적으로 실행하는 것이 좋음.

- 디스크의 논리적, 물리적인 오류를 검사하고 수정.


2. 디스크 조각 모음.

- 디스크에 단편화 ( FRAGMENTATION ) 되어 저장된 파일들을 모아서 디스크를 최적화.

- 단편화를 제거하여 디스크의 수행 속도를 높여줌.

- 총 용량을 늘려주는 것은 아니다.

- 실행 중인 프로그램을 종료해야 작업을 수행할 수 있음.




# 인쇄


1. 스풀 SPOOL

- 장치의 이용 효율을 높이기 위해 저속의 입출력 장치와 중앙 처리 장치의 동작을 동시에 이루어지게 처리 형태.


2. 버퍼링 BUFFERING.

- 한 작업에 대한 입출력 및 연산을 수행.

- 송싱자와 수신자의 속도 차이를 해결하기 위해 사용.

# UNIX의 특징.


- C언어 기반으로 제작된 시분할 온라인 대화식 시스템.

- 확장성과 이식성이 우수하며, 계층적인 디렉토리 구조를 가짐.

- 멀티태스킹 운영체제.

- 다중 사용자 시스템.

- 네트워킹 및 시스템 보호 기능이 뛰어남.

- 분산 처리 방식을 지원.

- CUI 환경을 제공함.




# UNIX의 구성.


1. 커널 KERNEL.

- UNIX의 가장 핵심적인 부분으로, 항상 주기억 장치에 상주하여 시스템의 자원관리.

- 프로세스 관리, 입출력 관리, 파일 관리, 메모리 관리, 시스템 호출, 프로세스 간 통신 관리.


2.셸 ( SHELL ) ! = CELL.

- 사용자와 UNIX 간의 인터페이스 역할.

- DOS의 COMMAND 역할.

- 단말기를 통해 명령을 입력 받은 후 커널이나 다른 유틸리티 프로그램을 실행시켜 해당 명령을 수행함.

- C SHELL = % , KORN SHELL, BOURNE SHELL = $.


3. 유틸리티 UTILITY.

- DOS의 외부 명령어에 해당됨.

- UNIX 시스템을 효과적으로 사용할 수 있도록 도와주는 응용 프로그램.


4. UNIX의 기억 장치 관리 기법.

- 주소 매핑 : 논리적인 주소를 물리적인 실제 주소와 1:1로 대응.

- 스와핑 : 보조 기억 장치의 남은 공간에 존재하는 프로그램의 일부를 주기억 장치로 가져오는 관리 기법.

- 페이징 : 주기억 장치와 보조 기억 장치 사이에 데이터 이동이 있을때, 페이지 단위로 처리.




# UNIX의 명령어.


1. 기본 명령어.

- login : 자신의 ID와 password를 입력.

- passwd : 비밀번호 설정 및 변경.

- logout : 유닉스 작업을 종료. ( exit, ctrl + D )

- who : 현재 사용자 이름을 표시.

- date : 현재 날짜 표시.

- time : command의 실행 시간 표시.

- man : 명령어들에 대한 도움말.

- ping : 네트워크 상의 연결 상태 점검.


2. 파일 및 디렉토리 관련 명령어.

- chmod : 디렉토리 속성 변경.

- is : 디렉토리의 파일 표시.

- cp : 파일 복사.

- mv : 이름 변경.

- pwd : 현재 디렉토리의 경로 표시.

- rm : 파일 삭제.

- cat : 파일의 내용 표시.

- cd : 디렉토리 경로 변경.

- more : 화면 단위로 내용 출력.

- mkdir : 디렉토리 생성.

- rmdir : 디렉토리 제거.



3.시스템 및 파일 관련 명령어.

- kill : 강제 중지.

- ps : 프로세스 상태보기.

- sleep : 일시 중지.

- shutdown : 체계적으로 종료.

- open : 파일 열기.

- close : 파일 닫기.

- finger : 사용자 정보 조회.

- chown : 소유권 변경.

- tar : 파일을 하나로 묶음.

- wc : 바이트 수 세기.

- tar의 옵션 : c = 생성, x = 추출, tr = 추가, t = 테이블보기, v = 정보 출력 추가, z = 압축하여 백업.




# 프로그램 편집기.


1. ed : 줄 단위 편집기.

2. ex : ed를 확장한 줄 단위 편집기.

3. vi : 화면 단위 편집기. ( 가장 많이 사용 )

4. emacs : vs와 유사하나 Ctrl과 Meta( Alt )를 사용.




# i - node


UNIX 파일에 대한 정보를 규정하는 자료 구조.

파일의 이름, 크기, 소유자, 파일의 종류, 파일의 위치 등에 대한 정보를 가짐.

DOS의 FAT와 유사한 개념.


i - node의 정보.

- 파일 소유자 및 그룹 식별 번호.

- 데이터 블록의 주소.

- 파일이 지정하는 디렉토리 수.

- 파일 생성 시간 및 변경 시간.

- 파일 소유자의 식별 번호.

- 파일의 종류와 크기.

- 파일 링크 수와 속성.

- 파일이 가장 최근에 사용된 시간.
