# 프로토콜( Protocal )의 정의


데이터의 송수신을 위해 필요한 일련의 절차나 규범의 집합.


1. 프로토콜의 기본요소

- 구문 ( Syntax ) : 데이터의 형식, 부호화, 신호 레벨 등.

- 의미 ( Semantics ) : 개체와 개체 간의 조정. 에러를 관리하기 위한 제어 정보.

- 순서 ( Timing ) : 메세지의 순서 제어, 통신 속도 제어.


2. 프로토콜의 종류.

- BSC ( Binary Synchronous Communication ) 프로토콜 : 문자 방식의 프로토콜.

- DDCM ( Digital Data Communication Message ) 프로토콜 : 바이트 방식의 프로토콜.

- SDLC ( Synchrounous Data Link Control ) 프로토콜 : 비트 방식의 프로토콜.

- HDLC ( High - level Data Link Control ) 프로토콜 : 비트 방식의 프로토콜.

OSI 7계층 중 데이터링크에서 사용.




# OSI 참조 모델 7계층.


1. 하위층

- 물리 계층 ( 1계층 ) : 기계적, 전기적, 물리적 절차를 규정.

- 데이터 링크 계층 ( 2계층 ) : 인접 개방형 시스템 간의 정보 전송 및 오류 제어.

- 네트워크 계층 ( 3계층 ) : 정보교환, 중계 기능, 경로 선정, 유통 제어 등.

- 전송 계층 ( 4계층 ) : 송수신 시스템 간의 논리적 안정 및 균등한 서비스 제공.


2. 상위층

- 세션 계층 ( 5계층 ) : 응용 프로세스 간의 연결 접속 및 동기 제어 기능.

- 표현 계층 ( 6계층 ) : 부호 교환, 암호화, 해독, 압축 등.

- 응용 계층 ( 7계층 ) : 응용 프로세스 간의 정보 교환 및 전자 사서함, 파일 전송 등.


3. 응용계층 추가 설명.

- OSI 참조 모델의 최상위 레벨로, 특정한 서비스( HTTP, FTP, 데이터베이스, 전자 사서함 등 )를 제공.

- 응용 프로그램과의 인터페이스 기능 및 통신을 수행.

# 정보 통신망의 구성 형태와 특징.


1. 스타형.

- 중앙 컴퓨터와 단말기들이 1:1로 연결되어 있는 형태로, 가장 기본적인 형태.

- 회선 교환 방식에 적합하다.


2. 트리형.

- 중앙 컴퓨터와 단말기가 연결되고 그 단말이게 또 다른 단말기가 연결되는 형태.

- 제어가 간단하고, 통신 선로가 가장 짧고, 단방향 전송에 적합하다.


3. 링형

- 서로 이웃하는 것끼리만 연결된 상태이다.

- LAN에서 가장 많이 사용.


4. 버스형.

- 한 통신 회선에 여러 대의 단말기가 접속되는 형태.

- 노드의 독립성을 보장.


5. 망 Mesh 형.

- 모든 단말기와 단말기들을 통신 회선으로 연결시킨 상태이다.

- 통신 회선의 전체 길이가 가장 길다.

- 통신 회선의 최소 링크 수 : n(n-1)/2.




# 회선 교환망 Circuit Switching Network.


컴퓨터와 단말기, 컴퓨터 간 통신 회선을 설정하여 데이터를 교환하는 방식이다.

Point - to - Point 방식으로, 통신회선을 고정적으로 할당함.

고정된 대역폭 전송 방식, 코드 변환 불가능.

실시간 대화용 방식의 응용 가능.

데이터를 전송하지 않을 때에도 회선이 점유되므로 자원의 낭비가 있음.

음성 전화 시스템, 원격 측정 입력과 같은 연속적인 흐름을 유지하는 데이터 교환에 적절함.




# 패킷 교환망 Packet Switching Network.


전송할 데이터를 송신측에서 패킷이라는 일정한 길이로 구분하여 보내는 방식.

회선 이용률이 가장 높고 전송량 제어와 전송 속도 변환이 쉬움.

각종 VAN 제공이 용이, 부가 서비스 제공.

대량의 데이터 전송시 전송 지연 발생.

공중 교환 데이터망 PSDN에서 대부분 사용하며, PAD가 필요함.

라우터 : 패킷의 경로 설정 기능, 네트워크 계층.




# 메세지 교환망 Message Switching Network


수신한 데이터를 중앙에서 축적, 처리하는 방식이다.

하나 또는 복수 개의 터미널로 전송.

노드 내의 메모리 축적 기능으로 회선 사용률을 높일 수 있음.

하나의 채널을 여러 메시지가 공유할 수 있어서 선로의 효율을 증대시킬 수 있음.

오류 제어, 코드 변환, 전송하는 메시지 길이에 제약이 없음.

응답 시간이 느리므로 대화형으로는 응용이 불가함.

전보, 전자우편, 컴퓨터 파일 등에 적절함.




# 근거리 통신망 ( LAN : Local Area Network )


1. LAN의 특징.

- 광대역 전송 매체의 사용으로 고속 통신이 가능하며 에러율이 낮음.

- 채널의 공동 사용으로 인해서 라우팅이 필요 없음.

- 음성, 화성, 데이터 등의 종합적인 정보 전송이 가능.

- 정보 기기의 재배치 및 확장성이 우수함.

- 방송 형태의 이용이 가능.

- 자원의 공유와 데이터의 일관성.


2. LAN의 구성 요소.

- 브리지 : 개방형 시스템 간의 상호 접속을 하기 위한 장비.

- 라우터 : 두 개 이상의 서브 네트워크와 서브 네트워크 계층을 결합하는 장치.

- 게이트웨이 : 다른 두 개의 프로토콜 구조를 가지는 7계층 간을 결합하는 데 사용.




# 광역 통신망 ( WAN : Wide Area Network )


다국적 기업 또는 상호 유대 관계가 있는 기관을 LAN으로 상호 연결시킨 망.

망이 폐쇄적인 양상이 있기 때문에 전문성과 안전성이 좋음.




# 종합 정보 통신망 ( ISDN : Integrated Services Digital Network )


음성 및 데이터나 화상 등 다양한 형태의 통신 서비스를 하나의 통신망인 디지털 통신망으로 제공하는 종합 통신망.


1. ISDN 서비스

- 텔레커뮤니케이션 서비스 : 모든 서비스가 포함되는 서비스의 총칭.

- 베어러 서비스 : 기본적인 전송의 통신 서비스. OSI 참조 모델의 1~3계층 기능만을 제공.

- 텔레 서비스 : 고도의 총체적 서비스를 제공. 사용자 입장에서 통신 서비스. OSI 상위 계층의 특성으로 규정.

- 부가 서비스 : 베어러, 텔레 서비스 외의 사용자를 위한 서비스.



# 비대칭 디지털 가입자 회선. ( ADSL : Asymmetric Digital Subscriber Line )


설치되어 있는 전화 회선을 통해 높은 대역폭으로 디지털 정보를 전송하는 기술.

대부분 채널을 사용자측으로 내려보내는 하향 전송에 사용하고,

사용자로부터 받는 정보에는 적게 할당되는 비대칭형 구조.

비대칭 구조이므로 쌍방향 서비스 혀ㅑㅇ태의 경우 서비스 효율이 떨어짐.

# 뉴 미디어의 종류.


1. 비디오 텍스 ( Videotex )

- TV 수상기와 전화망을 이용한 형태로, 다양한 서비스를 제공하는 유선 통신 뉴 미디어.

- 축적된 화상 정보의 데이터베이스로부터 각종 정보 검색 및 예약 업무, 홈 뱅킹 등 다양한 서비스 제공.

- 전화 + TV 수상기 + 데이터 뱅크.


2. 텔레텍스트 ( Teletext )

- TV 전파를 활용하여 다른 신호를 겹쳐서 정보를 제공하는 서비스.

- 방송 시스템 + 데이터 뱅크 + TV 수상기.

- TV 화면과 화면 사이의 수직 귀선 시간을 이용하여 정보를 한 방향으로만 전송.


3. 화상 시스템 ( VCS : Video Conference System )

- 화상 및 음향 등을 통해 회의 진행이 가능한 시스템.


4. OFDM ( 직교 주파수 분할 다중 전송 방식 )

- 서료 직교성의 속성을 지닌 여러 부 반송파를 이용하여 다중화하는 방식.


5. HDTV

- 주사선 수 1,125개이다.



# 멀티미디어( Multimedia ) 의 정의.


다양한 매체를 통해 정보를 전당하는 ' 다중 매체 '이다.

컴퓨터와 영상 매체 또는 방송과의 결합으로, 즉각적인 정보의 전달이 이루어짐.

양방향 정보 전달 가능. 압축 기술 사용.

CD-ROM과 DVD 등의 대용량 저장 매체가 이용.



# 정지 영상 데이터.


1. BMP : 비트맵 방식으로 표현. 압축하지 않아 화질은 좋으나 용량이 커짐.

2. GIF : - 비손실 압축을 하기 때문에 이미지의 손상은 없지만, 압축률이 좋지 않음.

- 256색까지만 표현할 수 있지만 배경을 투명하게 하거나 애니메이션 효과를 줄 수 있음.

3. JPEG : - 표준화 규격이다.

- 비손실 압축, 손실 압축 모두 지원. 주로 손실 압축 방식이 사용됨.

4. PNG : - GIF와 JPEG의 장점만을 조합하여 만든 형식.

- 투명한 배경, 애니메이션 효과를 지원하고 높은 압축률을 지원.



# 동영상 데이터.


1. MPEG : 영상 음성 등 다른 음향까지 압축하는 기술에 관한 표준화 규격.

2. VFW : 움직이는 영상을 캡처하는 기능, 영상의 압축을 소프트웨어만으로 가능.

3. DVI : 많은 양의 영상과 음향 데이터를 압축하여 CD-ROM에 기록 가능.

4. AVI : 소프트웨어를 이용하는 비디오 압축 방식. 빠르다.

5. MOV : 애플 사에서 만든 동영상 파일의 표준. Quick Time For Windows 프로그램이 필요.

6. ASF : 스트리밍 방식을 지원.

7. DivX : 고화질 파일 압축 형식, 비표준 동영상 파일 형식 ( 확장자 .AVI를 사용 )



# 사운드.


1. WAV : 직접 재생이 가능한 사운드 파일. 사람의 음성과 자연 음향 표현 가능.

2. MIDI : 일종의 통신 규약. 사람의 음성이나 자연음을 재생할 수 없음. ( 용량이 작음 )

3. MP3 : 전용 플레이어가 필욧한 파일. 1/12정도로 압축 가능.
